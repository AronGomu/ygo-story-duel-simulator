Fresh checkout

git clone git@github.com:AronGomu/ygo-story-duel-simulator.git
cd ygo-story-duel-simulator
npm ci
npm run assets:mvp
npm run check
npm run dev

npm ci

Installs exact lockfile dev/runtime deps. Node engine requirement: >=24.0.0.

npm run assets:mvp

Acquires vendored core, upstream DB/scripts/strings, image archive, generated snapshots. Resumable; rerun after network failure.

npm run check

Runs complete local release gates. First run installs no browsers automatically; Playwright browser binaries must exist.

npm run dev

Starts Vite. Dev middleware serves verified runtime files under configured base with no-store caching.

Quick launchers

Windows: download-mvp-assets.cmd. macOS/Linux: ./download-mvp-assets.sh. Both set project cwd, accept same flags as unified command.

Useful asset flags
npm run assets:mvp -- --offline
npm run assets:mvp -- --concurrency 8 --requests-per-second 12
npm run assets:mvp -- --force-images

--offline requires populated caches/archive. Default image concurrency/rate: 18; hard max rate: 20 req/s.

Verify environment incrementally

  1. Identity: node --version → 24+; npm --version; git --version.
  2. Generated status: generated/mvp-assets-status.json must say ready.
  3. Vendor: npm run vendor:verify.
  4. Data: npm run assets:verify + npm run snapshot:verify.
  5. Static quality: npm run format:check && npm run lint && npm run typecheck.
  6. Fast behavior: npm run test:unit.
  7. Real core: npm run test:integration.
  8. Browser package: npm run build, then npm run test:e2e.
Vite prerequisite. vite.config.ts reads generated/runtime/current/manifest.json immediately. Missing generated assets can fail config loading before dev server starts.

Mental model in 90 seconds

Rules

Core owns truth. UI never calculates legality. Worker protocol maps engine prompts to opaque choices.

Isolation

Worker owns secrets/handles. Main thread sees immutable redacted state and current human prompt.

Presentation

Svelte is complete UI. Phaser adds spatial visuals; failure cannot block duel controls.

Data

One compatibility snapshot. Engine/catalog/scripts/strings/images are pinned, hashed, activated together.

Reliability

Replace, do not trust uncertain state. Hung/broken Worker gets terminated and recreated.

Evidence

Headless first. Real-WASM fixtures/transcripts prove game support before UI polish.

Safe change recipes

Add a supported engine prompt family
  1. Add/verify engine message classification and stable fixture.
  2. Add domain PromptKind/choice shape only if existing model cannot express it.
  3. Map raw msg to opaque choices in PromptRegistry.
  4. Encode/validate response, retaining raw indexes only in registry binding.
  5. Add client-side prompt selection rules if multi-select/sum/order semantics differ.
  6. Add accessible control family in PromptControls.svelte.
  7. Add policy branch so opponent can always answer reachable prompt.
  8. Update exhaustive switches and event/contract parsers.
  9. Write unit fixture, component test, programmed real-WASM scenario, keyboard E2E only if browser path newly distinct.
Add field presentation for existing event
  1. Keep event domain-level; no raw core values.
  2. Map event to PresentationCommand with reduced-motion duration.
  3. Render cancellable effect in DuelScene.present.
  4. Ensure applying snapshot remains idempotent; event is cosmetic.
  5. Ensure restart/dispose cancels timers/tweens and releases refs.
  6. Test command mapping and bridge disposal/failure.
Change public duel state
  1. Define privacy requirement first.
  2. Change source contract and exact-key runtime parser together.
  3. Update projector and invariants.
  4. Update event parser privacy checks.
  5. Update store/UI/field consumers.
  6. Add structured-clone/round-trip tests, projection fixture, hidden-information test.
  7. Run Node Worker integration and browser event-capture assertions.
Update pinned assets or engine
  1. Use isolated branch; update refs/version/integrity intentionally.
  2. Regenerate full asset snapshot and runtime snapshot.
  3. Review count/protocol/active closure drift; never accept blind trace update.
  4. Run all protocol classification + programmed real-WASM suites.
  5. Run private build, independent verifier, reproducibility check, browser matrix.
  6. Archive exact manifests with release candidate; keep previous snapshot available.
  7. Repeat legal/distribution review when source/image posture changes.
Fix bug
  1. Classify layer: UI, projection, protocol, core/data, asset, storage, lifecycle.
  2. Capture fastest evidence: Worker event, downloaded trace, failing fixture, cache/manifest digest.
  3. Write smallest deterministic test reproducing failure before fix.
  4. Fix owning layer only. Do not hide projection/protocol bug in Phaser/UI.
  5. Run direct suite, then boundary integration, then broader gate based on affected contracts.

Worker log vocabulary

  • duel.worker.command.* — receive/reject/fail/complete/skip.
  • duel.worker.event.* — event dispatch/post failure.
  • duel.worker.detached — attachment teardown.
  • duel.worker.node.* — Node Worker bootstrap/thread boundary.
  • duel.worker.logging.failed — logger fallback failure.
  • duel.client.worker.* — main-thread creation, boundary failure, disposal/replacement.
  • duel.field.* — lazy Phaser startup/texture failures.
  • duel.app.* — storage activation/cleanup and destroy failures.

Diagnostic file

Request after duel result/failure. File contains app/browser/build metadata, active/fallback snapshot IDs, image cache counts, core version, upstream revisions, production seed, ordered trace. Treat as sensitive. Reproduce with exact snapshot + seed + ordered responses in authorized local workflow.

Common traps

Import engine on main thread

Breaks isolation and packaging. Only OcgCoreAdapter.ts imports vendored package.

Use Phaser as state truth

Sprites are disposable projections. Query public state/store instead.

Hide instead of redact

Card back does not protect code in object. Remove identity before Worker boundary.

Fetch inside core callback

Callbacks are synchronous. Preload all card/script data into Worker maps.

Expose raw indexes

UI uses namespaced opaque prompt/choice IDs. Raw response indexes remain private binding data.

Reuse uncertain Worker

Cleanup failure means terminate + replace. Never continue on possibly live handle.

Run Vite before assets

Config needs generated manifest. Complete asset baseline first.

Update traces to make CI green

Understand semantic drift. Fixtures are compatibility evidence.

Package full data archive

Production includes recursively resolved active closure only.

Publish private dist

Current images/content lack approved redistribution posture.

Ready-to-change checklist

  • ✓ Full setup succeeds; generated status ready.
  • ✓ You can explain main-thread vs Worker ownership.
  • ✓ You know owning architecture doc and direct tests for task.
  • ✓ You traced producer + boundary parser + consumer for contract changes.
  • ✓ You selected smallest reproduction/test before implementation.
  • ✓ You know whether change affects snapshots, private packaging, privacy, replay evidence.
