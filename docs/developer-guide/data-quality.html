Asset acquisition and transformation

Unified command: npm run assets:mvp. Offline regeneration: npm run assets:mvp -- --offline. Run is cross-process locked; final status becomes ready only after all verifiers pass.

Atomic snapshot identity

Runtime snapshot ID

Content-derived from verified asset manifest content digest + vendored engine manifest digest. Identifies rules/data/script compatibility unit.

runtimeSnapshotId = sha256(
  runtimeAssetContentSha256,
  engineVendorManifestSha256
)

Activation snapshot ID

Separates browser activation when image subset changes without runtime change.

activationId = sha256(JSON.stringify({
  runtimeSnapshotId,
  activeImageManifestSha256
}))

Why two IDs

Engine/data/scripts define duel compatibility. Active image manifest defines UI artifact set and legal distribution packaging. Combining them for browser activation prevents image-only release from colliding with existing persisted runtime record.

Browser data and persistence

Activation transaction

  1. Validate exact input shape, digests, runtime/image receipt consistency.
  2. Stage immutable candidate unless identical record exists.
  3. Verify receipts exactly equal required set.
  4. Open read-write transaction over snapshots + pointers.
  5. Validate active/previous pointer targets and generations.
  6. Compare actual active ID/generation to caller expectation.
  7. Move active→previous, candidate→active, update pointers to next generation.
  8. Delete stale previous after atomic commit.
Cross-tab safety. App combines Web Lock around activation/cache cleanup with IndexedDB compare-and-swap. Lock serializes cooperating tabs; CAS rejects stale actors even if lock unavailable/logic changes.

Production package flow

  1. vendor:verify proves vendored engine package/file hashes.
  2. snapshot:verify proves current runtime root and all generated inputs.
  3. vite build --mode private compiles Svelte, dedicated module Worker, sync engine adapter, lazy Phaser chunk.
  4. Vite runtime plugin copies only recursively resolved active-deck closure, active preset images, engine WASM/manifests, licenses.
  5. Unapproved image posture writes PRIVATE_DEPLOYMENT_ONLY.txt; ordinary non-private packaging refuses.
  6. build:verify independently rehashes source vs dist, exact file sets, content-derived IDs, engine chunk, Worker digest embedding, activation ID, licenses, forbidden Node markers, size budgets.
  7. build:reproducible creates two isolated builds and requires identical hashes.

Security and licensing boundaries

Security

Trusted local inputs only

  • Pinned project-provided Lua, decks, manifests.
  • Path containment + realpath checks in dev/build asset serving.
  • Strict manifest/payload bounds.
  • Dedicated Worker termination boundary.
  • Hidden identities removed before IPC.
  • No user scripts/decks or multiplayer authority in MVP.
Distribution

Public deployment blocked

  • Embedded core + CardScripts carry AGPL obligations.
  • BabelCDB redistribution terms need separate review.
  • Artwork/content/trademarks need permission.
  • Image provider access does not grant redistribution.
  • Current active manifest explicitly says redistributionApproved: false.
  • Build marker says keep artifact private.
Do not publish dist/ publicly. Existing build is intentionally private-only until source availability, DB terms, artwork/content permissions receive authorized review.
