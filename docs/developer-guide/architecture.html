Worker contracts

Public protocol intentionally small. High-cardinality game protocol never crosses boundary directly.

Commands: main → Worker

type DuelCommand =
  | { type: "initialize" }
  | { type: "startDuel"; duelId: DuelId }
  | { type: "respond"; promptId: PromptId;
      choiceIds: readonly ChoiceId[] }
  | { type: "surrender" }
  | { type: "requestDiagnostics" }
  | { type: "dispose" };

Events: Worker → main

type DuelWorkerEvent =
  | Ready | Loading | State
  | PresentationEvent | Prompt
  | Result | Diagnostics | Error
  | Disposed;

Every event parsed by parseDuelWorkerEvent before store. Parser enforces clone bounds, exact keys, dense arrays, prompt player, public-state privacy.

Source: src/duel/contracts/duel-worker-event.ts

Boundary defense layers

  1. Sender validation. Client passes every command through parseDuelCommand before postMessage.
  2. Receiver validation. Worker parses untrusted message again before queueing.
  3. Session validation. Runtime checks initialized/active state and preset ID.
  4. Prompt validation. Only current prompt ID resolves; selection count/order/sum/place rules checked.
  5. Engine validation. Encoded response goes through vendored core.
  6. Outbound validation. Client parses every event; malformed/privacy-breaking event terminates Worker.
  7. Context validation. Store rejects events from old Worker/session generations.

State machines

Worker lifecycle

spawned → initialize queued → resources ready → one duel active → result/dispose → handle destroyed. Runtime can retain last bounded trace after session ends. Uncertain cleanup sets replacementRequired.

Snapshot lifecycle

staged → receipt-verified → active → previous → removed. Activation uses expected active ID + generation in one IndexedDB transaction. One previous snapshot retained for rollback.

Privacy path

Critical invariant. Opponent hidden card identity must never cross Worker boundary. Rendering a card back is insufficient if code/name remains in JS object.
  1. Raw core message may contain card code and field location inside Worker.
  2. DuelStateProjector applies visibility policy. Opponent hand gets handCount, but public hand array is empty.
  3. When visible card becomes hidden, projector rotates public instance ID and deletes code.
  4. PromptRegistry redacts prompt card code/name/description when viewer cannot see identity.
  5. Outbound event parser rechecks public state: opponent hand identity array must remain empty.
  6. Svelte/Phaser receive hidden cards without identity; Phaser selects card-back texture from hidden.
  7. E2E captures real Worker events and asserts opponent hand objects remain empty.
